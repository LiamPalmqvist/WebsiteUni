Github + Git integration for CI435

This guide aims to enable you to create, manage and utilise repositories in Git for MacOS, Windows and Linux.

Installation

To get started, install Brew if you haven’t already. This is imperative as it allows you to install Git and its dependencies.

If you want to check that you have Brew installed, in the terminal enter
brew --version

Git

To install Git, open your terminal program of choice and enter
brew install git

To test Git out and make sure it’s been installed properly in your currently open terminal enter
git --version

Git credential manager

This will allow you to sign into Git using Github to edit your repositories.

To install the credential manager, open the terminal and enter
brew tap microsoft/git
brew install --cask git-credential-manager-core

You won’t need to do anything more with this as when you sign try to commit to your future repository the credential manager will automatically boot up and ask you to sign in.

Setting up the repository

Now that we’ve set up Git and the login manager, we can start to set up the files and folders necessary to use them

Start by making a new public repository in Github

Take note of the URL you’ve typed in as we’ll be using it later

Setting up the folder

Now, in your file browser, create the folder that will contain your code. If you’ve already got a folder with code that you wish to upload to GitHub, the process is the same.

Take note of the address and enter terminal. In your terminal, cd into the directory you just created. This works on OneDrive too, so don’t worry about that.

In terminal, in the directory you wish to create your repository in, enter
git init
This will create an empty repository for you to fill and configure.

If you already have files in the folder, enter
git add .
This will add the files in the folder to the repository.

To link the local git repository to the one you created on GitHub enter
git remote add origin (URL)
Afterwards, we want to set up our main branch. To do this, enter
git branch -M main
This will also put us on the main branch of the repository.

to check the status of the main branch you just linked, enter
git status on branch main

The local Git repository should now be linked to the main branch of your GitHub repository

Pushing and committing files

Git won’t automatically commit and push files to GitHub when you make changes so you have to do it yourself

Committing changes

To commit changes to a repository you’ll need to make sure that all the files you want to include are included in the Git repository. To include all new files, enter

git add .

This will do as it did before and add all the files in the folder to your repository to be included in commits and pushes.

to commit your changes now, you’ll need to provide a changeling. This can be as short as one word or as long as a paragraph, include bullet points, etc.

This can be done easily with the git commit -am "Comment" command

Pushing to GitHub

This is easy as well. All you need to do to push your most recent changes is use
git push -u origin main
This pushes all of your most recent changes in the commit command to the main branch of your GitHub repository.

Signing into Git with GitHub

If it’s your first time using Git to manipulate an offline repo, you’ll be asked to sign into your GitHub account as soon as you try to commit or push for the first time. This is why we installed the Git Credential Manager earlier as it makes it easier for you to sign in.

This method is also used as the previous sign in method which used the terminal to sign in was insecure and is now deprecated.

This is fairly simple to do. Just follow the steps on the pop-up window that appears.

brighton.domains

Creating the reference and linking the repository

Once we’ve created our git repository and linked it to our GitHub one, we can now start to download our files to the brighton.domains file system.

NOTE: Linking our repository does NOT give brighton.domains permission to edit what is in the repository, only to download it.

To do this, make sure your public_html folder is empty and create a git link in the Git Version Control page on brighton.domains

After, enter your url information, give it a name, and set the repository path to be in /home/EX###/public_html. Make sure “Clone a Repository” is turned ON and then hit Create

This should have linked the repository to the public_html folder in your file system within brighton.domains

Updating the website

Once the repo is linked, it won’t update unless asked to. This means we have to go into the settings for the repository link and update it to reflect the most recent changes.

Click Manage to enter the settings for the repo

Click Pull or Deploy to enter the deploy settings and then click Update from Remote to pull the most recent changes from the repository and update the site.
